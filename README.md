# מסע כתב רש"י — LMS ללימוד כתב רש"י

גרסה חדשה ועצמאית של אתר תרגול כתב רש"י (שהיה ב-Google Sites + H5P),
בנויה מאפס עם מעקב התקדמות לתלמידים ומפת חום למורים.

**כתובת חיה:** https://chepti.com/rashi/

## מה יש כאן

- **צד תלמיד** — מסע של 9 יחידות בסדר פדגוגי (מהאותיות הדומות לכתב רגיל ועד כל האותיות).
  8 סוגי משחקים: כרטיסי היכרות, כרטיסי תמלול, חידון קריאה, תפזורת (כולל מצב "הפוך"),
  התאמה בגרירה, זיכרון, סיפור עם קו מפריד נגרר (רש"י/רגיל), סידור אלף-בית, וציור לפי הוראות.
  אף פעילות לא משתמשת באות שטרם נלמדה. קונפטי, כוכבים, ונעילה הדרגתית.
- **צד מורה** — הרשמה עצמית, יצירת כיתות עם קוד מספרי בן 6 ספרות,
  קישור הצטרפות (`#/join/123456`) שהילדים לוחצים בלי להקליד קוד,
  צפייה בכל שלבי המשחק בלי נעילה, ומפת חום לפי אותיות.
  תלמידים נכנסים עם קישור או קוד + כינוי — בלי סיסמאות.
- **מעקב פר-אות** — כל תשובה בכל משחק נזקפת לאותיות הרלוונטיות (נכון/שגוי),
  ורמת השליטה (0–100) משוקללת עם משקל-יתר לטעויות וביטחון גדל עם כמות הנתונים.

## מבנה

```
app/    — פרונטאנד: Vite + React + TypeScript (RTL, עברית)
  src/data/     — אותיות, טיפוסים, ותוכנית הלימודים (units.ts — כל התוכן)
  src/games/    — רכיבי המשחקים + GameHost
  src/views/    — מסכים: כניסה, מפת מסע, יחידה, התקדמות, לוח מורה
  src/lib/      — api, חישוב שליטה, עזרי התקדמות
  public/fonts/ — Noto Rashi Hebrew + Heebo (וריאביליים, self-hosted)
api/    — בקאנד: PHP + SQLite (בלי תלות ב-MySQL)
  db.php        — סכימה, טוקנים חתומים (HMAC), תיקיית נתונים
  teacher.php   — הרשמה/התחברות/גוגל, כיתות, מפת חום
  student.php   — הצטרפות בקוד, דיווח פעילות, שליפת התקדמות
```

## פיתוח מקומי

```bash
cd app && npm install && npm run dev
```

מצב "תרגול חופשי" עובד בלי שרת (localStorage). ה-API דורש PHP:
`php -S localhost:8090` מתיקיית השורש (ה-proxy ב-vite.config מפנה אליו).

## פריסה

```powershell
cd app; npm run build
scp -r -F "T:\.ssh\config" app/dist/* hostinger:~/public_html/rashi/
scp -r -F "T:\.ssh\config" api hostinger:~/public_html/rashi/
```

בסיס הנתונים נוצר אוטומטית ב-`~/rashi_data/rashi.db` (מחוץ ל-web root).

## כניסת גוגל למורים (אופציונלי)

צרו OAuth Client ID בקונסולת Google Cloud והדביקו ב-`api/config.php` בשרת.
בלי זה — הרשמה עם אימייל וסיסמה עובדת רגיל.
