## מה יש כאן

### תלמידים
- מסע של **9 יחידות** בסדר פדגוגי (אותיות דומות לכתב רגיל ← א' ← ח' ← ב' ← ש' ← צ'/ץ' ← ד' ← סיפור ← סיכום)
- **מפת מסע** גלילה אנכית עם רקע מאויר, תחנות על השביל וכוכבים; נפתחת **מתחתית** (תחילת המסע)
- **סוגי משחק:** היכרות (היפוך כרטיס), כרטיסי תמלול, חידון, תפזורת, התאמה, זיכרון, סיפור עם קו מפריד, סידור אלף-בית, ציור לפי הוראות
- נעילה הדרגתית — אין פעילות שמשתמשת באות שטרם נלמדה
- כניסה עם **קוד כיתה מספרי** או קישור `#/join/123456` (כינוי + אימוג'י, בלי סיסמה)
- **תרגול חופשי** בלי כיתה (נשמר ב־localStorage), כולל אפשרות «מסלול חופשי»
- רקעי עמודים מאוירים בשקיפות; דפי כניסה עם רקע מלא

### מורים
- הרשמה / התחברות (אימייל+סיסמה; כניסת גוגל אופציונלית)
- יצירת כיתות עם קוד בן 6 ספרות וקישור הצטרפות
- **מפת חום** לפי אותיות (שליטה 0–100)
- **צפייה בכל שלבי המשחק** בלי נעילה
- עורך מיקומי תחנות: `#/path-edit`
- כפתור **משוב** (mailto) בלוח המורה ובתרגול חופשי

### טכני
- פרונט: Vite + React + TypeScript, RTL, ניתוב ב־hash (`#/map`, `#/play/...`)
- בקאנד: PHP + SQLite (בלי MySQL)
- פונטים self-hosted: **Noto Rashi Hebrew** (כתב רש"י) + **Heebo** (כתב מרובע / דפוס)
- אותו תו Unicode מוצג בשני פונטים — אין המרת תווים, רק החלפת `font-family`

## פיתוח מקומי

דרישות: Node.js 18+, PHP 8+ (ל־API).

```bash
# טרמינל 1 — פרונט
cd app
npm install
npm run dev
```

```bash
# טרמינל 2 — API (משורש הריפו)
php -S localhost:8090
```

ה־proxy ב־`vite.config.ts` מפנה `/rashi/api` → `http://localhost:8090/api`.

- **תרגול חופשי** עובד גם בלי PHP (הכול ב־localStorage).
- כניסת כיתה / לוח מורה דורשים את שרת ה־PHP.

```bash
cd app && npm run build && npm run preview
```

## פריסה

האתר החי יושב תחת `public_html/rashi/`:

```
public_html/rashi/     ← תוכן app/dist/*
public_html/rashi/api/ ← תיקיית api/
```

מסד הנתונים: `~/rashi_data/rashi.db` (מחוץ ל־web root כשאפשר).

### דחיפה טיפוסית

```powershell
cd app
npm run build
scp -r -F "T:\.ssh\config" dist/* hostinger:~/public_html/rashi/
scp -r -F "T:\.ssh\config" ..\api hostinger:~/public_html/rashi/
```

(התאימו את Host `hostinger` ב־`~/.ssh/config`.)

### כניסת גוגל למורים (אופציונלי)

1. OAuth Client ID ב־Google Cloud (סוג Web).
2. Authorized JavaScript origin של האתר.
3. Client ID ב־`api/config.php` בשרת.

בלי זה — הרשמה עם אימייל וסיסמה עובדת כרגיל.

## איך להעתיק ולהתאים לעצמכם

הפרויקט בנוי כך שאפשר לשכפל אותו ללימוד נושא אחר / מותג אחר / נתיב אחר.

### עקרון חשוב

| שכבה | מה משתנה בדרך כלל |
|------|-------------------|
| **תוכן** | `units.ts` (+ לפעמים `letters.ts`) — **רוב העבודה** |
| **תצוגת «כתב מיוחד»** | פונט + class `.rashi-font` ב־`styles.css` |
| **מיתוג / נתיב** | מחרוזות UI + `base: '/…/'` |
| **מנוע המשחקים** | בדרך כלל **לא** צריך לגעת — `GameHost` והמשחקים כבר משווים שני כתבים |

### 1. שיבוט

```bash
git clone https://github.com/chepti/Rashi.git MyCopy
cd MyCopy/app
npm install
```

מומלץ: מאגר GitHub נפרד + תיקיית פריסה נפרדת בשרת (למשל `/ktav/` ליד `/rashi/`).

### 2. נתיב הבסיס (חובה אם לא `/rashi/`)

החליפו `/rashi/` בנתיב שלכם בכל הקבצים הבאים:

| קובץ | מה לשנות |
|------|-----------|
| `app/vite.config.ts` | `base` + ה־proxy |
| `app/src/lib/api.ts` | `const API = '/…/api'` |
| `app/src/lib/pathLayout.ts` | נתיבי רקע המפה |
| `app/src/ui/PageShell.tsx` | נתיבי רקעי העמודים |
| `app/src/views/Landing.tsx`, `Join.tsx`, `Teacher.tsx` | רקעים |
| `app/index.html` | preload לפונטים/רקעים |
| `app/src/styles.css` | `@font-face` → `url('/…/fonts/…')` |

לשורש האתר: `base: '/'` ונתיבים בלי קידומת תיקייה.

### 3. תוכן הלימוד — `units.ts`

כל היחידות והפעילויות בקובץ אחד: **`app/src/data/units.ts`**.

כל יחידה כוללת:
- `id`, `title`, `subtitle`, `icon`
- `newLetters` — האותיות **החדשות** ביחידה הזו
- `activities[]` — פעילויות מסוגים שב־`types.ts`

**כלל זהב:** בכל פעילות יופיעו רק אותיות שכבר נלמדו (`newLetters` של היחידות הקודמות + הנוכחית). אין אכיפה אוטומטית בקוד — זה באחריות כותב התוכן.

מטא-נתונים לרמזים / אותיות קשות / סופיות: `app/src/data/letters.ts`.

### 4. מיתוג ומסכים

| מה | איפה |
|----|------|
| כותרת דף | `app/index.html` |
| דף כניסה | `app/src/views/Landing.tsx` |
| הצטרפות | `app/src/views/Join.tsx` |
| לוח מורה | `app/src/views/Teacher.tsx` |
| צבעים | `app/src/styles.css` (`:root`) |
| משוב + קרדיט | `app/src/ui/Feedback.tsx` |
| רקעים | `app/public/bg-*.webp` |
| שם חבילה | `app/package.json` |

### 5. מפת המסע

- רקע: `app/public/bg-journey.webp` (+ `bg-journey-tiny.webp`)
- מיקומים: `app/src/lib/pathLayout.ts` → `STATION_POS`
- **מספר הנקודות ב־`STATION_POS` = מספר כל הפעילויות** בכל היחידות
- עריכה ויזואלית: התחברות מורה → `#/path-edit` → שמירת המערך חזרה לקוד
- צבעי יחידות: `UNIT_COLORS` באותו קובץ (הוסיפו צבעים אם יש יותר יחידות)

### 6. הפרדה מעותק אחר באותו דומיין

אם `/rashi/` ו־`/ktav/` רצים יחד:

| מה | למה |
|----|-----|
| תיקיית פריסה נפרדת | קבצים לא נדרסים |
| `api/db.php` → קובץ DB אחר (למשל `ktav.db`) | נתוני כיתות לא מתערבבים |
| מפתחות `localStorage` (`rashi_*` → `ktav_*`) | סשן אורח/מורה לא מתנגש |

קבצים עם מפתחות `rashi_*`: `api.ts`, `pathLayout.ts`, `JourneyMap.tsx`, `JourneyTrail.tsx`, `PlayView.tsx`, `sound.ts`, `App.tsx`.

### 7. רשימת בדיקה אחרי העתקה

1. `base` + כל נתיבי `/rashi/` עודכנו  
2. `npm run build` עובר  
3. דף הכניסה נטען בנתיב החדש  
4. תרגול חופשי עובד  
5. מורה + כיתה + הצטרפות תלמיד מול ה־API  
6. המפה נפתחת מלמטה; מספר תחנות = מספר פעילויות  
7. קרדיט / מייל משוב ב־`Feedback.tsx`

## דוגמה: עותק KTAV (כתב דפוס ↔ כתב עגול)

רוצים מסע נפרד שמתרגל בין **אותיות מרובעות (דפוס)** ל־**כתב עגול** — עם סדר אותיות שונה לגמרי מרש"י.

### מה נשאר כמו שהוא
- מנוע המשחקים (`games/`), ההתקדמות, המפה, לוח המורה, ה־API
- הרעיון: אותו Unicode, שני פונטים (מרובע = Heebo; עגול = פונט ktav חדש במקום Noto Rashi)

### מה חובה לשנות

#### א. פונט
1. שמו קובץ `ktav-var.woff2` (או דומה) ב־`app/public/fonts/`
2. ב־`styles.css`: `@font-face` בשם `'Ktav'` ו־class `.ktav-font`
3. החליפו בכל הפרויקט `rashi-font` → `ktav-font`
4. עדכנו preload ב־`index.html`

#### ב. סדר למידה חדש → שכתוב `units.ts`

סדר הקבוצות (כל קבוצה = יחידה עם `newLetters`):

| # | אותיות חדשות |
|---|--------------|
| 1 | י, ו, נ *(וסופית ן עם נ)* |
| 2 | נ *(כבר נלמד)*, ר, ה → בפועל `newLetters`: ר, ה |
| 3 | ח, ק, ת |
| 4 | ס, כ *(וסופית ך)*, א |
| 5 | ב, ג, ד |
| 6 | מ, ם |
| 7 | ע, ט, ש |
| 8 | ז |
| 9 | ל |
| 10 | פ, ף |
| 11 | צ, ץ |

**כן — זה משפיע על כל התוכן:** כל מילה, שאלה, תפזורת, `fillPool` וזוגות חייבים להיבנות מחדש כך שישתמשו **רק** באותיות שנלמדו עד אותה יחידה. אי אפשר «לסדר מחדש» את יחידות רש"י הקיימות בלי לשכתב מילים.

מומלץ לכל יחידה: `intro` → תרגול (`flashcards` / `match` / `memory`) → סיכום (`quiz` / `wordsearch`).  
בסוף: יחידת גמר עם `order` (אלף-בית) וחידון משולב — אופציונלי.

#### ג. `letters.ts`
עדכנו `hint` / `hard` להבדלים בין מרובע לעגול (לא לרש"י).

#### ד. מיתוג ופריסה
- שם: למשל «מסע כתב» / «מסע KTAV»
- `base: '/ktav/'`, DB `ktav.db`, מפתחות `ktav_*`
- `STATION_POS` + `UNIT_COLORS` לפי מספר הפעילויות/יחידות החדש
- קרדיט ומשוב ב־`Feedback.tsx`

#### ה. הערות על הקבוצות ששלחתם
- **יחידה 1 «יון»** → אותיות י, ו, נ (+ ן)
- **יחידה 2 «נ, ר, ה»** — נ כבר ביחידה 1; ביחידה 2 מוסיפים ר, ה (אפשר לחזק נ בפעילויות)
- סופיות (`ן ך ם ף ץ`) כדאי להכניס באותה יחידה עם האות הבסיסית שלהן

### היקף עבודה משוער
| משימה | היקף |
|--------|------|
| נתיב `/ktav/` + מיתוג + פונט | קטן–בינוני |
| שכתוב מלא של `units.ts` (11 יחידות) | **עיקרי** |
| התאמת מפה / תחנות | בינוני |
| DB + localStorage נפרדים | קטן |

אחרי שהעותק מוכן — פריסה ל־`public_html/ktav/` באותו אופן כמו רש"י.
